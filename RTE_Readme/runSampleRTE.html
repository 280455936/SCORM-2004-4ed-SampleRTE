Running the SRTE

Instructions

These instructions are for running the SRTE on a Windows platform. Windows is the only platform on which the software has been tested and for which instructions are provided.

To Run the SRTE:

Ensure that your default browser is Microsoft Internet Explorer 6.0 or greater and then start the apache-tomcat-5.5.20 servlet container and Web server that is provided with this installation by:

  1. Clicking the shortcut at Start Menu -> All Programs -> ADL -> SCORM_2004_4th_Edition_Sample_Run-Time_Environment_version_1.1.1 -> Start Server. A command prompt window will appear and show several lines of textual information. This window will remain open until you stop the server.
  2. Clicking the Start Menu -> All Programs -> ADL -> SCORM_2004_4th_Edition_Sample_Run-Time_Environment_version_1.1.1 -> SCORM 2004 4th Edition Sample Run-Time Environment version 1.1.1 will launch the SRTE Main Page.
  3. Log in using
    • username: "admin"
    • password: "admin"

Online help pages are available once the SRTE is started. "Help" links will typically be located on the top right of each page.

To close the SRTE, first close the browser in which it is running. To stop the apache-tomcat-5.5.20 servlet container and Web server, click Start Menu -> All Programs -> ADL -> SCORM_2004_4th_Edition_Sample_Run-Time_Environment_version_1.1.1 -> Stop Server.

To return the SRTE to a "clean" state, log into the SRTE and from the home page choose "Clear Database". This will remove all of the imported course files as well as any information associated with them. It will also remove all user registration information and all Global Objective information.

Course Navigation User Interface

In addition to any User Interface (UI) controls provided within course content, the SRTE provides several mechanisms for navigating through a course that complies with Section 2.1.7 of the SCORM 2004 4th Ed. Testing Requirements. These mechanisms include the navigation buttons in the top frame of the SRTE, such as the Quit, Suspend, Previous, and Continue buttons, and the Table of Contents provided in the left-hand frame.

Navigation Buttons

The navigation buttons provided by the SRTE in the top frame will be visible when they are not explicitly "hidden" by an activity via the "hideLMSUI" element within the manifest. When an activity explicitly hides any of the buttons, they will not appear in the top frame.

The following section describes the behavior of the navigation buttons when they are not hidden:

  • Quit - This button will issue an Exit All navigation request to the SRTE's Sequencer; the SRTE will end a user's current attempt on a course. The user's progress and course status information for the current attempt will no longer be available if the user makes another attempt on the course.
  • Suspend - This button will issue a Suspend All navigation request to the SRTE's Sequencer; the SRTE will end a user's current attempt on a course and will also 'bookmark' the user's position within the course. The user's progress and course status information will persist. When the user starts the course again, the course will resume with the same content where they triggered the suspend.
  • Previous - This button will issue a Previous navigation request to the SRTE's Sequencer, resulting in launching the activity identified prior to the current activity in the course's activity tree.
  • Continue - This button will issue a Continue navigation request to the SRTE's Sequencer, resulting in launching the activity identified after the current activity in the course's activity tree.

Table of Contents

A table of contents (ToC) will be provided by the SRTE when a course includes Choice equal to "true" anywhere within its activity tree. When a course includes Choice equal to "false" at every level of its activity tree, the table of contents will not appear.

The following describes the items in the ToC when Choice is set to "true" in the course:

  • Selectable - Items that are selectable will be indicated with black text and the cursor will change as the user moves an input device over them.
  • Not Selectable - Items that are not selectable will be indicated by gray text or black text and a cursor that does not change when the user moves an input device over them.
