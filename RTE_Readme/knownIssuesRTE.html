Additional Information

Known Issues
  1. If a package manifest contains references to Sharable Content Objects (SCOs) that are located on an external web site, a cross-domain JavaScript security error will occur. This may prevent the SCOs from communicating with the SRTE provided API.

  2. The Internet Explorer browser must be set to check for a new page every visit to the page. To configure the browser:
    1. Open a Microsoft Internet Explorer Browser
    2. Click on "Tools" then "Internet Options"
    3. On the "General" tab, if using Internet Explorer 6 under "Temporary Internet Files" or under the "Browsing History" section if using Internet Explorer 7, click Settings
    4. Select the "Every visit to the page" radio button if it is not already selected
    5. Click OK and then OK again.

  3. Switching between different versions of the SRTE may cause an Applet Error if Applet Caching is enabled. If this occurs, follow these steps to correct the issue:
    • For Java SE 6
      1. Open the Windows "Control Panel"
      2. Double click the "Java" icon
      3. Click the "General" tab at the top of the dialog box
      4. Under the "Temporary Internet Files" section, click Settings
      5. Ensure that the check box labeled "Keep temporary files on my computer" is unchecked.
