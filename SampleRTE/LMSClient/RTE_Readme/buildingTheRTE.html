Building the SRTE

Note: You are not a required to build the SRTE before using it. It will install as a fully operational unit. You only have to build the SRTE if the source code has been changed.

Complete the SRTE full installation instructions before compiling the SRTE. The SRTE build file depends on a full apache-tomcat-5.5.20 install, along with resources provided in the installation of the SRTE. When the SRTE is built, it overwrites the previously installed SRTE. To revert back to the original SRTE, execute the un-install and then rerun the downloaded self-extracting executable.

Apache Ant must be installed before building the SRTE. For information about and downloads of Ant, refer to http://ant.apache.org/.
To build the SRTE:

  1. Open a command prompt and navigate to the directory where the SRTE is installed (Default: C:\ADL\SCORM_2004_4th_Ed_Sample_RTE_V1.1.1)
  2. Navigate to the Sample_RTE\source directory
  3. At the prompt type "ant".

The XML Ant build files will compile all Java class and jar files needed by the SRTE. In addition the files will be installed to apache-tomcat. ( Note: Restart the Tomcat server for changes to take effect. This completes the compilation of the SRTE.

Source File Locations

  • All JSP, HTML, and JavaScript files are located at <drive>:\<install_directory>\Sample_RTE\source\SampleRTE\LMSClient.
  • The SRTE Java files are located in their respective package locations. For example the data model classes are located at "org\adl\datamodels". The root of the Java source directory is <drive>:\<install_directory>\Sample_RTE\source\src. There you will find the build script and the root "org" directory.
