Revisions
Changes from SCORM 2004 4th Ed. Sample RTE Version 1.1 to SCORM 2004 4th Ed. Sample RTE Version 1.1.1
  • The SRTE course import process was updated to enforce the following rules:
    • Parents of a given element are correct in both name and XML namespace.
  • The SRTE was also updated to:
    • Correctly display user status information when no courses exist.
    • Provide the user with the options to reset or sort courses.
Changes from SCORM 2004 4th Ed. Sample RTE Version 1.0 to SCORM 2004 4th Ed. Sample RTE Version 1.1
  • The SRTE course import process was updated to enforce the following rules:
    • Children of the <imsss:sequencing> element must exist only under an <imsss:sequencing> element.
  • The SRTE Sequencer was updated to:
    • Determine if the root was suspended before deleting the Data Store objects.
    • Check the state of each activity before performing post condition evaluations.
    • Correctly process "adl.nav.request_valid" GetValue() and SetValue() requests.
    • Store and return "incomplete" when "not attempted" is set on secondary objectives and later retrieved via a shared global objective.
  • The SRTE was also updated to:
    • Handle all course management activities from one central Home Page. Course import, registration, launching, and status viewing were combined into one central location which also contains links to administrative options and user management options.
Changes from SCORM 2004 3rd Edition Sample RTE Version 1.0.2 to SCORM 2004 4th Ed. Sample RTE Version 1.0
  • The SRTE course import process was updated to enforce the following rules:
    • If the "xsi:schemaLocation" attribute is used, then every namespace used in the XML instance must be listed in that attribute.
    • If the "xsi:schemaLocation" attribute is used, the actual locations of the attribute must be checked, even if they are not in the root.
    • The <adlnav:presentation> element must have a parent that has an "identifierref" attribute that refers to a resource with "scormtype" attribute equal to "sco" or "asset".
    • The <adlcp:timeLimitAction>, <adlcp:completionThreshold>, and <adlcp:dataFromLMS> elements shall only be used with <item> elements that reference resources with the "scormtype" attribute equal to "sco".
    • All extension elements must be valid against the controlling documents(XSDs) provided.
    • The correct multiplicity on the <ruleCondition> element beneath any of the <preConditionRule>,<postConditionRule>, <exitConditionRule> elements must be one and only one.
    • An "ID" attribute cannot be allowed on a <sequencing> element that is a child of an <item> or <organization> element.
    • The <targetObjectiveID> element must not contain only white space characters.
    • Packages that contain (sub)manifests in the <manifest> element (or its children) will fail to import when Validation is enabled. In cases that a (sub)manifest exists, but isn't referenced from the <manifest> element, it will be ignored during execution.
    • Packages containing <resource> "href" attributes that reference external URLs will fail to import.
    • Packages that contain sequencing information (The use of "flow = false" and "choice = false" Control Modes on the package root with no reachable children) which will result in no reachable activities at the time of course launch will fail to import.
    • Objective IDs which differ only in case must be treated as unique objective IDs. (ex: "OBJ1" is a different ID than "obj1")
  • The SRTE Sequencer was updated to:
    • If an <item> contains a <imsss:sequencing> element that has an IDRef to a <imsss:sequencing> element in an <imsss:sequencingCollection> then the elements in the sequencingCollection are now properly merged into the sequencing information for the item.
    • Behave correctly regarding the use of the "constrainChoice = true" attribute on the <adlseq:constrainedChoiceConsiderations> element on the root of the course.
    • Correctly initializes a cluster activity and process the default Rollup Rules.
    • Correctly handle Activity status "unknown" so that this invalid state does not cause a Not "Satisfied" or Not "Completed" rule from triggering in the LMS.
  • The SRTE was also updated to:
    • Properly process Extended Rollup based on status changes to the secondary objectives.
    • Allow the "Quit" button to properly persist " completion status" and "success status" data.
    • Directly launch the single leaf Activity in content packages that contain only a single SCO.
    • Disable all Navigation Controls while the course content is being loaded or unloaded.
    • Correctly process the whitespace component of identifier, indentifierref, ID, and IDRef attributes based on the defined datatypes of those attributes.
